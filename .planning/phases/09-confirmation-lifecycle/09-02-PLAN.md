| requester client → confirm_donation RPC | Requester-supplied donor_code crosses into the privileged RPC (server re-checks ownership + participant) |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-09-02-01 | Spoofing | Scanning/typing a non-participant or guessed code | mitigate | Client `/^[A-Z2-7]{5}$/` filter is convenience only; authority is the server RPC participant check (09-01) returning generic `invalid_code` (D-06) |
| T-09-02-02 | Information Disclosure | Error messages revealing whether a code is a real donor | mitigate | Single generic "Invalid or unrecognized code" for both unknown and non-participant; specific message only for the already-confirmed (already-known-valid) case (D-06) |
