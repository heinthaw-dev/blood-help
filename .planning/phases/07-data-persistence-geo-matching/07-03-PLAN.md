<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| form input → DB write | User-entered name/phone/blood type cross into Postgres; must be validated and coarsened |
| client → blood_requests insert | Untrusted client controls the insert; RLS + unique index are the server-side guards |
| navigator.geolocation → DB | Raw GPS must be coarsened before it leaves the client (PRIV-03) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-07 | Information Disclosure | Raw (uncoarsened) GPS written to donors/blood_requests | mitigate | coarsenCoordinates() called before onSave in DonorProfileSetup; CreateRequest already coarsens; handlers receive pre-coarsened coords only (PRIV-03, D-10/D-11) |
| T-07-08 | Tampering | Double-submit creating two active requests | mitigate | UI-gate hides Request CTA when hasOpenRequest (D-16, wired from hydrated active request); DB unique index backstop surfaces as 23505 → duplicate AlertDialog (D-17) |
| T-07-09 | Elevation of Privilege | Writing a donors/profiles row for another user | mitigate | Upsert keyed by uid = session user id; donors RLS WITH CHECK profile_id=auth.uid() (plan 01) rejects cross-user writes |
| T-07-10 | Denial of Service / data integrity | Unhandled write error crashing the app (no error boundaries exist) | mitigate | All DB calls resolve to {data,error}; errors logged + surfaced via AlertDialog (D-18), never thrown (CLAUDE.md no-throw) |
| T-07-SC | Tampering | npm/pip/cargo installs | accept | No new packages this phase (07-RESEARCH §Package Legitimacy Audit — none added); no install task present |
</threat_model>
