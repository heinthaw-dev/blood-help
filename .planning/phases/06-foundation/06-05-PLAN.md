## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-17 | Spoofing | signInAnonymously called when session already exists | mitigate | getSession() called first; signInAnonymously only called when session === null (Pitfall 1 prevention — PATTERNS.md shows wrong order; plan action text overrides it) |
| T-06-18 | Information Disclosure | handleVerified phone lookup in profiles | mitigate | Query uses `.eq('phone', phone)` parameterized via Supabase JS — no SQL injection risk; returns only id column |
| T-06-19 | Information Disclosure | GPS coordinates written to DB (Phase 7+) | mitigate | coarsenCoordinates() added to geolocation.ts (D-10, D-11); raw GeoResult.lat/lng must never be passed to DB write functions |
| T-06-20 | Denial of Service | signInAnonymously rate limit | accept | Supabase rate-limits anonymous sign-ins per IP; error fallback shows PhoneEntry normally — acceptable degradation |
| T-06-21 | Tampering | localStorage JWT tampered by malicious script | transfer | Supabase server validates JWT signature; tampered token rejected at Supabase API level — not an app-layer concern |
| T-06-SC | Tampering | npm/pip/cargo installs | accept | No new package installs in this plan — only source file edits |
