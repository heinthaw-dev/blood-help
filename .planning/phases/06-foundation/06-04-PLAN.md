| .env.local → Vite build | Anon key is embedded in the client bundle — this is expected; anon key is public by design in Supabase |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-13 | Information Disclosure | VITE_SUPABASE_ANON_KEY in client bundle | accept | Supabase anon key is intentionally public — it identifies the project, not the user; all data access gated by RLS using the user's JWT |
| T-06-14 | Tampering | .env.local committed to git | mitigate | .gitignore contains *.local; verify before writing; if git status shows .env.local as untracked, add it to .gitignore explicitly and re-verify — never stage or commit |
| T-06-15 | Information Disclosure | Service role key vs anon key confusion | mitigate | Only anon key goes in VITE_SUPABASE_ANON_KEY — service role key MUST never appear in .env.local or client code |
| T-06-16 | Elevation of Privilege | Multiple createClient() instances | mitigate | src/lib/supabase.ts is the sole createClient call; JSDoc enforces this; no other file may call createClient |
| T-06-SC | Tampering | npm install @supabase/supabase-js | mitigate | Package Legitimacy Audit approved in RESEARCH.md: official Supabase client, 5+ yrs, no postinstall script, verified via npm registry and official docs |
