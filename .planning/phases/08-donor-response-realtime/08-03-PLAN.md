<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> responders_for_request RPC | The requester's browser calls the RPC with the active request_id (the owner guard is the gate) |
| client -> request_responses Postgres Changes stream | The browser subscribes filtered to one request_id; RLS-on-stream gates delivery per JWT |
| effect lifecycle -> realtime socket | A leaked/unclosed channel is an authorized socket left open after navigation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-09 | Information Disclosure | A non-owner subscribes to another request's request_responses stream (filter is client-set and spoofable) | mitigate | Postgres Changes applies the existing Phase 6 SELECT policy per subscriber JWT; only the parent request's requester_id (or the donor) receives events. A spoofed filter for another request connects but delivers nothing. Plan 01 only added the table to the publication; the policy is unchanged. |
| T-08-10 | Information Disclosure | Donor phone leaked via the realtime payload | mitigate | The effect NEVER reads phone from the postgres_changes payload (payload carries only request_responses columns). Phone arrives ONLY via the owner-guarded responders_for_request RPC refetch (D-11). |
| T-08-11 | Information Disclosure | Requester calls responders_for_request for a request they do not own | mitigate | The RPC owner guard (Plan 01, T-08-01) returns 0 rows for a non-owner; RequestLive only ever passes its own activeRequestId (threaded from the owner's hydrated request). |
| T-08-12 | Information Disclosure | Leaked authorized socket after navigating away from RequestLive | mitigate | Effect cleanup returns supabase.removeChannel(channel) (Pattern 7); a stable channel name rr:${requestId} prevents duplicate channels on re-mount (Pitfall 5). |
| T-08-13 | Spoofing (UX-correctness) | Transparency line overstates reality ("alerted") implying a push that never happened | mitigate | D-09: copy is a truthful "can see your request" count of compatible donors within radius; the word "alerted" is forbidden. |
| T-08-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs this phase (realtime ships inside the already-vetted @supabase/supabase-js). No legitimacy checkpoint required. |
</threat_model>
