<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → PostgREST RPC | Authenticated browser calls `responders_for_request`; the request_id argument is attacker-controllable |
| client → Realtime socket | Authenticated browser subscribes to `request_responses` Postgres Changes; the JWT identifies the subscriber |
| RPC body → tables (SECURITY DEFINER) | The function runs with elevated rights; search_path and schema qualification are the containment boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Information Disclosure | `responders_for_request` called with another user's request_id (enumeration of others' responders/phones) | mitigate | Owner guard in the body: `r.requester_id = (SELECT auth.uid())` + `IF NOT FOUND THEN RETURN` → 0 rows, identical empty result whether the request exists or not (no existence signal). GRANT to `authenticated` only. |
| T-08-02 | Information Disclosure | Donor phone leaked via the realtime stream payload | mitigate | The publication streams only `request_responses` columns (no phone). Phone is returned ONLY by the owner-guarded RPC; Plan 03 refetches via the RPC and never reads phone from the payload. |
| T-08-03 | Information Disclosure | Non-owner subscribes to another request's `request_responses` Postgres Changes stream | mitigate | Postgres Changes applies the existing Phase 6 SELECT policy per subscriber JWT; only the parent request's `requester_id` (or the donor) receives events. Non-owner subscription connects but receives nothing. This plan only ADDS the table to the publication; it does not loosen the policy. |
| T-08-04 | Elevation of Privilege | SECURITY DEFINER search_path hijack | mitigate | `SET search_path = ''` + every PostGIS / table identifier schema-qualified (`extensions.`, `public.`), carried verbatim from the deployed requests_within_radius RPC. |
| T-08-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs this phase (realtime ships inside the already-vetted @supabase/supabase-js). No legitimacy checkpoint required. |
</threat_model>
