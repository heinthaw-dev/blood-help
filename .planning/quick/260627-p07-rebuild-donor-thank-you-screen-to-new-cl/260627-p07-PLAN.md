# Quick Task 260627-p07 — Rebuild Donor Thank You screen

## Goal

Rebuild `src/screens/DonorThankYou.tsx` from scratch to match the new Claude Design
("Donor Thank You.dc.html", project 3aee6b2b). The screen changes from a static
thank-you with one red CTA into a screen that **owns the push-notification opt-in**,
with three states (idle/enable · needsInstall · enabled) and a demoted secondary
"Continue to home" link. Wire the enable action to the app's real push infra, not
the design's dummy `Notification.requestPermission` stub.

## Tasks

### Task 1 — Rebuild DonorThankYou.tsx + wire real push, thread supabaseId from App

- **files:** `src/screens/DonorThankYou.tsx`, `src/App.tsx`
- **action:**
  - Add `supabaseId: string | null` to `DonorThankYouProps`; pass `user.supabaseId`
    from `App.tsx` where `donor-thankyou` renders.
  - Keep hero block: `ScreenHeader variant="brand" align="center"`, heart icon
    (circle 88px / icon 40px), headline (30/600), subheadline (18/500), blood-type
    `Badge` row, warm message (15/1.75, max-width 300). Main content flows from the
    top (`padding 24px 0 12px`), not vertically centered.
  - Notification opt-in block (`flex:none`) with three states:
    1. **idle/canEnable:** reuse `Card` (surface + 1px border + `shadow-card`) with a
       bell-icon chip + enable copy, and a full-width red `Button tone="primary"`
       ("Turn on alerts", bell icon).
    2. **needsInstall** (iOS Safari, not standalone — web push unavailable): replace
       the button with an "Add to Home Screen" guidance row (upload icon + copy).
    3. **enabled:** `Card` with `--color-success-tint` background, success check
       circle, "Alerts are on" in `--color-success`.
  - "Continue to home" demotes to a quiet transparent secondary text link
    (text-secondary → text-primary on hover).
  - Push logic (real infra): `import { pushSupported, registerPushToken } from '../lib/push'`.
    - On mount: `enabled` ← `Notification.permission === 'granted'`; if already
      granted and `supabaseId`, silently re-register the token (preserves the old
      `maybeAskPush` granted-branch behavior). Detect `needsInstall` from
      iOS-Safari-not-standalone UA check. Never auto-request permission on load.
    - `onEnable` (tap only): `await registerPushToken(supabaseId)`; flip to enabled
      on `'granted'`.
  - Remove the now-redundant `maybeAskPush(uid)` call in `handleSaveDonor` (the
    thank-you screen now owns this prompt). Leave `maybeAskPush` + the Home/RequestLive
    AlertDialog push prompts untouched (still used by `handlePosted`).
  - Bilingual EN/MY copy (Burmese-first) for all new strings; tokens only (no hex
    except `#fff` per house style); reuse `Badge`, `Card`, `Button`.
- **verify:** `npm run build` (tsc -b) and `npm run lint` both pass.
- **done:** Screen renders all three push states with correct tokens/copy; donor is
  no longer double-prompted on save; build + lint green.
