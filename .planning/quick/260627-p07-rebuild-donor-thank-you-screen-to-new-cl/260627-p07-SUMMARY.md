# Quick Task 260627-p07 — Summary

## What changed

Rebuilt `src/screens/DonorThankYou.tsx` from scratch to match the new Claude Design
("Donor Thank You.dc.html", project 3aee6b2b). The screen is no longer a static
thank-you with a single red CTA — it now **owns the push-notification opt-in**.

### `src/screens/DonorThankYou.tsx` (rebuilt)
- Hero block preserved via shared components: `ScreenHeader variant="brand" align="center"`,
  heart icon (circle 88px / icon 40px), headline (30/600), subheadline (18/500),
  blood-type `Badge` row, warm message (15/1.75, max-width 300). Main content now
  flows from the top (`padding 24px 0 12px`) instead of being vertically centered.
- New notification opt-in block with three states:
  1. **idle/canEnable** — `Card` (surface + 1px border + `shadow-card`) with a bell
     chip + prompt copy, and a full-width red `Button tone="primary"` ("Turn on alerts").
  2. **needsInstall** — iOS-Safari-tab (web push unavailable): the button is replaced
     by an "Add to Home Screen" guidance row (upload icon + copy).
  3. **enabled** — `Card` with `--color-success-tint` background, success check circle,
     "Alerts are on" in `--color-success`.
- "Continue to home" demoted from the primary red Button to a quiet transparent
  secondary text link (text-secondary → text-primary on hover).
- Wired to the **real** push infra (`pushSupported`, `registerPushToken` from
  `lib/push.ts`) — not the design's dummy `Notification.requestPermission` stub.
  - Initial `enabled`/`needsInstall` derived via `useState` lazy initializers from
    `Notification.permission` + an iOS-Safari-tab UA check (client-only SPA).
  - A mount-only effect silently re-registers the FCM token when permission was
    already granted before this screen (preserves the old `maybeAskPush` granted
    branch). Permission is requested **only on tap**, never on load.

### `src/App.tsx`
- Threaded `supabaseId={user.supabaseId}` into `<DonorThankYou>`.
- Removed the now-redundant `maybeAskPush(uid)` call in `handleSaveDonor` (the
  thank-you screen owns the prompt now) — avoids double-prompting the donor.
  `maybeAskPush` + the Home/RequestLive AlertDialog push prompts are untouched
  (still used by `handlePosted`).

## Reuse / consistency
- Reused `Card`, `Button` (tone=primary, shadow-cta/height/hover baked in), `Badge`,
  `ScreenHeader`. No parallel components introduced.
- Tokens only; the only literal color is `#fff` for icon strokes (house-style exception).
