# Quick Task 260625-v8m — Frontend FCM Wiring (Part B)

## Goal

Land the frontend Firebase Cloud Messaging (FCM) web-push wiring for the Blood Help
PWA so the app can register device tokens, receive foreground/background pushes, and
deep-link from a notification tap — as the precursor to a Vercel deploy. This pulls the
**FCM push feature forward from its v3.0 deferral** (see STATE.md Deferred Items).

## Context discovered at execution time

The bulk of this work was **already present in the working tree but uncommitted**
(firebase installed; `src/lib/firebase.ts`, `src/lib/push.ts`,
`src/firebase-messaging-sw.js`, PWA icons, `vite.config.ts` VitePWA config, and the
App/Home/RequestLive UI wiring). The implementation uses `vite-plugin-pwa`'s
`injectManifest` strategy to emit a **single merged service worker** at the site root
(FCM background handler + Workbox precache), which satisfies spec §1
("single merged service worker").

So this task = **harden + checkpoint** the existing work, not greenfield build.
